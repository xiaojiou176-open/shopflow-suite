# Sidepanel Override — Shopflow UIUX Endgame R3

## Purpose

Side Panel is the main product surface.

## Must Have

- readiness at the top
- unmistakable primary route
- explicit operator next step
- visible claim boundary
- latest runnable output / proof hint
- progressive disclosure for evidence depth

## Must Not Have

- duplicate first-screen summaries
- default-open report walls
- equal-rank blocks from top to bottom
- governance or packet detail above product action hierarchy

## Layout

- first viewport should read:
  - readiness
  - primary route
  - claim boundary
  - operator next step
  - latest proof
- capability matrix and evidence stack may follow below

## Review Check

- if “what should I do now?” is not answerable in 3 seconds, sidepanel failed
